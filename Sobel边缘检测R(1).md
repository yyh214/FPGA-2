## 三、实验原理

### 3.1 UART 图像传输协议

本实验采用自定义 UART 图像传输协议，与后续上板实验保持一致，协议格式如下：

* ​**帧头**​（7 字节）：`0x55 0xAA width_l width_h height_l height_h 0x18`
  * `0x55 0xAA`：帧同步头，用于识别图像帧开始
  * `width_l/width_h`：图像宽度（128）的低 / 高字节
  * `height_l/height_h`：图像高度（72）的低 / 高字节
  * `0x18`：表示像素格式为 RGB888（每个像素 3 字节）
* ​**行头**​（4 字节）：`0x33 0xCC row_l row_h`
  * `0x33 0xCC`：行同步头，用于识别图像行开始
  * `row_l/row_h`：当前行号的低 / 高字节
* ​**像素数据**​：每行包含 128 个像素，每个像素 3 字节，顺序为 R、G、B

### 3.2 Sobel 边缘检测算法

Sobel 边缘检测分为 RGB 转灰度和边缘强度计算两个步骤。

#### 3.2.1 RGB 转灰度

采用符合人眼视觉特性的加权公式： **\$\$Gray = 0.299 \\times R + 0.587 \\times G + 0.114 \\times \$\$** 硬件实现中使用移位和加法近似计算，避免乘法器占用过多资源。

#### 3.2.2 Sobel 边缘强度计算

使用 3×3 卷积核分别计算水平和垂直方向的梯度：

* 水平梯度核 Gx：
  ```Plain
  -1  0  1
  -2  0  2
  -1  0  1
  ```
* 垂直梯度核 Gy：
  ```Plain
  -1 -2 -1
   0  0  0
   1  2  1
  ```

边缘强度为两个梯度的绝对值之和： **Edge = |Gx| + |Gy|**

### 3.3 系统数据流

实验系统采用流水线设计，数据流如下：

1. 输入 RGB 图像转换为 hex 格式字节流
2. UART 接收模块解析字节流
3. 图像帧解析模块提取有效像素数据
4. RGB 转灰度模块将彩色像素转换为灰度值
5. Sobel 核心模块通过 3×3 窗口计算边缘强度
6. 视频流输出模块将结果保存为 PGM/PNG 格式

​**UART 帧头接收与帧开始波形**​：

波形分析：

1. 复位信号`rst_n`在仿真开始后拉高，系统进入工作状态
2. UART 接收数据`rx_data`依次出现`0x55`、`0xAA`等帧头字节
3. 当完整帧头接收完成后，`frame_start`信号产生一个高脉冲，表示一帧图像开始处理
4. `uart_rx_line`信号为 UART 字节接收完成指示，每接收一个字节产生一个高脉冲

​**完整图像处理周期波形**​：

波形分析：

1. `rgb_valid`信号在像素数据有效时保持高电平，持续时间对应一帧图像的像素传输时间
2. `gray_valid`信号比`rgb_valid`延迟一个时钟周期，这是 RGB 转灰度模块的流水线延迟
3. `edge_valid`信号比`gray_valid`延迟 3 个时钟周期，这是 Sobel 核心模块的 3×3 窗口缓存延迟
4. 当`edge_frame_done`信号拉高时，表示一帧图像的 Sobel 边缘检测已经全部完成
5. 图像分辨率为 128×72，波形中`rgb_x`和`rgb_y`的最大值分别为`0x007F`（127）和`0x0047`（71），与图像分辨率一致

### 5.3 UART 帧格式与数据流说明

* ​**UART 帧格式**​：采用字节流传输，先传输 7 字节帧头，再逐行传输 4 字节行头和 128×3 字节像素数据。每个像素按 R、G、B 顺序传输，每行传输完成后传输下一行的行头，直到所有 72 行传输完毕。
* ​**Sobel 数据流**​：像素数据从 UART 接收后，经过帧解析模块提取 RGB 数据，送入灰度转换模块；灰度数据经过 3 行缓存形成 3×3 滑动窗口，送入 Sobel 核心计算边缘强度；最后输出边缘数据并在所有像素处理完成后生成`edge_frame_done`信号。整个流程采用流水线设计，模块间通过 valid 信号握手，保证数据传输的正确性。
