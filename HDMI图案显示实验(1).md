HDMI图案显示实验报告

## 一、实验目的

1. 掌握 HDMI 720p 显示时序的基本参数和工作原理
2. 理解`video_clock`时钟 IP 核和`rgb2dvi_0` TMDS 编码 IP 核的作用
3. 掌握 128×72 小分辨率图像映射到 1280×720 HDMI 显示区域的实现方法

## 三、实验原理

### 3.1 系统整体数据流

本实验采用纯 PL 端实现的 HDMI 显示方案，数据流如下：

1. 128×72 RGB 图像以 Verilog ROM 形式固化在 FPGA 内部
2. `hdmi_image_display`模块产生 720p 显示时序，同时从 ROM 中读取像素数据并进行 10 倍放大
3. `rgb2dvi_0` IP 核将并行 RGB 信号转换为 TMDS 串行差分信号
4. 差分信号通过 HDMI 接口输出到显示器显示

### 3.2 HDMI 720p 显示时序

HDMI 720p@60Hz 标准时序参数：

* 像素时钟频率：74.25MHz
* 水平分辨率：1280 像素
* 垂直分辨率：720 行
* 水平同步总周期：1650 像素
* 垂直同步总周期：750 行

`video_clock` IP 核用于生成精确的 74.25MHz 视频时钟，是 HDMI 显示正常工作的基础。

### 3.3 图像缩放原理

本实验采用**最近邻插值法**实现 10 倍图像放大：

* 水平缩放系数：`SCALE_X = 1280 / 128 = 10`
* 垂直缩放系数：`SCALE_Y = 720 / 72 = 10`
* 实现方式：将 ROM 中同一个像素数据重复输出 10×10 次，对应 HDMI 显示的 10 个像素点

### 3.4 Verilog ROM 像素格式

ROM 中每个像素采用 24 位 RGB888 格式存储，编码为`24'hRRGGBB`：

* 高 8 位：红色分量 (R)，范围 0x00\~0xFF
* 中 8 位：绿色分量 (G)，范围 0x00\~0xFF
* 低 8 位：蓝色分量 (B)，范围 0x00\~0xFF

## 四、实验步骤

1. **打开 Vivado 工程** 打开路径为`D:\Github\FPGA-course\zynq7020-image-processing\sobel_01_hdmi_pattern\sobel_01_hdmi_pattern.xpr`的工程，确认`top.v`为顶层模块。
2. **查看核心参数** 打开`hdmi_image_display.v`，确认显示参数：
   ```verilog
   H_ACTIVE = 1280    // HDMI水平有效像素
   V_ACTIVE = 720     // HDMI垂直有效行
   IMG_WIDTH = 128    // 输入图像宽度
   IMG_HEIGHT = 72    // 输入图像高度
   SCALE_X = 10       // 水平缩放系数
   SCALE_Y = 10       // 垂直缩放系数
   ```
3. **综合与实现** 在 Vivado 中依次执行：
   1. Run Synthesis（运行综合）
   2. Run Implementation（运行实现）
   3. Generate Bitstream（生成比特流）
4. **下载到开发板** 连接 JTAG 下载器和 HDMI 数据线，打开硬件管理器，将生成的`top.bit`文件下载到 ZYNQ7020 开发板。
5. **扩展实验** 修改`hdmi_image_display.v`，实现扩展功能：
   1. 调整图片显示位置
