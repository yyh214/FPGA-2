HDMI Sobel边缘检测实验报告

## 一、实验目的

1. 掌握 RGB888 彩色图像转换为 8 位灰度图像的硬件实现方法
2. 理解 Sobel 卷积运算的原理及边缘强度的计算过程
3. 掌握`edge_mem`帧缓存的作用及读写控制逻辑
4. 学会将固定图片的 Sobel 边缘检测结果通过 HDMI 接口放大显示

* 开发板：Xilinx ZYNQ7020 (xc7z020clg400-2)
* 开发工具：Vivado 2017.4

* 辅助设备：JTAG 下载器、HDMI 数据线

## 三、实验原理

### 3.1 系统整体数据流

本实验在 HDMI 固定图片显示的基础上集成了图像处理流水线，整体数据流如下：

1. 从`image_rom_128x72`中读取 128×72 RGB888 格式的固定图片
2. 经过`rgb_to_gray`模块转换为 8 位灰度图像
3. 送入`sobel_core`模块进行 3×3 卷积运算，输出 8 位边缘强度
4. 边缘结果写入`edge_mem`进行一帧缓存
5. `hdmi_sobel_display`模块从`edge_mem`中读取边缘数据，放大 10 倍后输出到 HDMI 显示器

### 3.2 RGB 转灰度算法

采用符合人眼视觉特性的加权公式： **\$\$Gray = 0.299 \\times R + 0.587 \\times G + 0.114 \\times \$\$** 硬件实现中使用移位和加法近似计算，避免乘法器占用过多资源。模块输入为`rgb_valid`和 24 位`rgb_data`，输出为`gray_valid`和 8 位`gray_data`。

### 3.3 Sobel 边缘检测算法

使用两个 3×3 卷积核分别计算水平和垂直方向的梯度：

* 水平梯度核 Gx：
  ```Plain
  -1  0  1
  -2  0  2
  -1  0  1
  ```
* 垂直梯度核 Gy：
  ```Plain
  -1 -2 -1
   0  0  0
   1  2  1
  ```

边缘强度为两个梯度的绝对值之和：**Edge = |Gx| + |Gy|**。模块输入为`gray_valid`和 8 位`gray_data`，输出为`edge_valid`和 8 位`edge_data`。

### 3.4 edge\_mem 帧缓存机制

`edge_mem`是一个 128×72×8bit 的单端口 RAM，用于缓存一帧完整的边缘检测结果。其作用包括：

1. 解决图像处理流水线延迟与 HDMI 显示时序的同步问题
2. 避免显示过程中出现图像撕裂现象
3. 实现图像处理与显示的解耦，提高系统稳定性

### 3.5 Sobel 结果到 HDMI 的映射

将 8 位边缘强度值同时赋值给 HDMI 输出的 R、G、B 三个通道，实现灰度边缘显示：

```verilog
hdmi_r = edge_data;
hdmi_g = edge_data;
hdmi_b = edge_data;
```

边缘强度越高，像素越亮，最终呈现黑底白边的效果。

### 5.6 关键数据流说明

1. ​**edge\_mem 写入时机**​：当`sobel_core`输出`edge_valid`为高电平时，将当前的`edge_data`写入`edge_mem`，写入地址由`wr_addr = img_y * IMG_WIDTH + img_x`计算得到。
2. ​**edge\_mem 读取时机**​：在 HDMI 显示的有效区域内，根据显示坐标计算对应的图像坐标`img_x = hcount / SCALE_X`、`img_y = vcount / SCALE_Y`，然后从`edge_mem`中读取对应的边缘数据。
3. ​**模块间握手**​：各个模块通过`valid`信号进行数据传输的握手，保证流水线的顺畅运行。
